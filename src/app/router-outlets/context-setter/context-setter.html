Context

A context component is one that can switch internal components (context) at runtime. Why would we need such a thing?

Keeps DOM Small

Switching context prevents bloat from hiding/showing different components, especially when they don't need to keep an internal state

Internal Tracking

You can maintain a single pointer as a variable to know where you are in a flow. This can reduce the number of if/switch statements.

Lazy Loading

You load the content when you need it; it doesn't have to be pre-rendered or preserved (though it can with multiple context components, if you wish).
